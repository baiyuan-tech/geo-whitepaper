## 為何錯開時間

- **個人帳號優先發布** — 工程內容更適合工程師個人 voice,先建立技術社群討論
- **24h / 48h 跨語系錯開** — LinkedIn 演算法對同帳號連發多語不友善;錯開讓每種語言都有獨立曝光週期
- **公司帳號 +5-7 天** — 等個人發文累積互動數後,公司頁轉貼或獨立發,提供「客戶角度」訊息(歡迎合作 / 接洽)
- **個人 + 公司分離訊息** — 個人聚焦「我做了什麼、踩了什麼雷」(工程語氣);公司聚焦「我們提供什麼、為何公開」(商業語氣)

## 內容遵守原則

✅ **以 DOI + 工程實質開頭**,不以行銷話術開頭
✅ **承認未解問題**(L3 cache、phase 升級、wiki fallback eval、ME metadata gating)
✅ **僅宣稱可驗證內容** — 16 章、50,000 字、Zenodo DOI、CC BY-NC 4.0、ORCID 全部可查驗
✅ **不使用「台灣第一個」「業界唯一」「全球首創」等未經第三方驗證宣稱**
✅ **連結正確**:DOI = `10.5281/zenodo.19994035`(concept,自動跳最新版),GitHub = `baiyuan-tech/geo-whitepaper`
